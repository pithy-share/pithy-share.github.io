# 《从0到1理解AI》— 大纲

> 一份"自己也能学懂"的 AI 科普教材。从神经元一路走到 agent，把两块第一性原理骨头讲透，其余现象都从它们推导出来。

## 目标与定位
- **读者**：有编程基础、零 AI 基础的自学者（也作为自己学习的笔记）
- **最终形态**：HTML 教材（分章节、可交互示例、代码高亮、图解）
- **不是**：纯学术教科书，也不是"30 分钟速成"的快餐

## 三条基调
1. **深度**：感性理解为主，原理讲透。只用小学数学（加减乘除、平均），不写矩阵 / 概率 / 求和号 / 偏导这些形式——核心是原理，不是形式
2. **重心**：原理精炼到"够用"，重点放在 LLM 工程应用（提示词 / 上下文 / harness / agent 工程）
3. **动手**：每章配几十行可跑 Python，概念 + 手感并重

## 两个核心原则（贯穿全书）
1. **第一性原理**：每个概念都追到最底层——它到底是什么、为什么这样设计。具体落点见下面两块"骨头"。
2. **简洁明了**：能一句话说清的绝不用两句；直觉优先、例子优先；砍掉一切凑字数的展开。

### 全书两块"骨头"（第一性原理的具体落点）
- **骨头 #1：学习 = 参数更新**（损失 → 梯度 → 反向传播 → 参数滑动）
- **骨头 #2：大模型 = "下一个 token 的概率函数"** `f(上下文) → 概率分布`

> 这两块不是某一章带过，而是**锚点章**——讲透后，后面所有现象（生成、采样、幻觉、温度、CoT、上下文）都从它们推导，让学习者"自己想明白为什么"，而不是背结论。

## Part A · 原理骨（广度"够用"，但骨头 #1 在此讲透）

### 第 1 章　机器到底怎么"学" — 🔒 第一性原理 #1：学习 = 参数更新
- 什么是"学习"：从经验里调参数，让误差变小
- 损失函数：怎么衡量"错得多离谱"
- 梯度下降：沿着下坡方向走（一维直觉 → 多维）
- 反向传播：链式法则怎么把误差一层层传回去（**此处给一点数学，讲透**）
- ▶ 动手：手写一维线性回归，亲眼看参数自己"滑"到对的位置

### 第 2 章　神经网络 — 把简单的东西拼成强大的
- 神经元 = 线性组合 + 非线性激活（为什么必须有激活：没它再多层也只是线性）
- 从一层到深度：深度为什么管用（特征的层次组合）
- 过拟合 / 泛化：模型为什么会"死记硬背"，怎么治
- ▶ 动手：numpy 手写两层网络做分类（几十行）

### 第 3 章　监督学习 — 给答案，学规律
- 监督学习 = 给 (输入, 标签)，学一个映射
- 分类 vs 回归；它的瓶颈：要大量标注
- 伏笔：监督学习是 RLHF 的一条腿，后面会接回来

### 第 4 章　强化学习 — 给奖励，学策略
- 监督 vs 强化：给答案 vs 给分数
- 核心词汇：智能体 / 环境 / 状态 / 动作 / 奖励 / 回报
- 探索 vs 利用；策略梯度（给一个最简公式：让"导致好结果的动作"以后更可能出现）
- ▶ 动手：网格世界，几行代码看 agent 自己学会走路
- 伏笔：这正是 RLHF 和"agent 自我进化"的内核

## Part B · LLM 核心（骨头 #2 在此讲透，并和 #1 接上）

### 第 5 章　大模型到底是什么 — 🔒 第一性原理 #2：大模型 = "下一个 token 的概率函数"
- 一句话定义：LLM 是函数 `f(上下文) → 下一个 token 的概率分布`
- token 是什么（分词）；这个函数有多大、它"记住"了什么（不是数据库，是压缩的世界知识）
- 从概率函数出发，所有现象都能解释：生成=采样、温度、幻觉、重复、上下文
- ▶ 动手：调 API 拿 logits，自己实现 temperature / top-k 采样

### 第 6 章　Transformer — 让"读懂上下文"变强的架构
- 之前 RNN 的困境：记不住远处
- 自注意力：每个词去"看"所有其他词、算相关性（**此处给 Q/K/V 公式，讲透**）
- 多头注意力、位置编码、残差 / 层归一（各自解决什么问题）
- Decoder-only：为什么 GPT 类模型只往前生成
- ▶ 动手：numpy 手写最小 self-attention（几十行）

### 第 7 章　训练 — 把概率函数"烤"进去（#1 和 #2 在此合流）
- 预训练 = 在海量文本上做"下一个词预测" = 一直在更新参数（呼应第 1 章）
- 损失：交叉熵（为什么这个损失对"概率分布"最自然）
- 规模律：为什么越大越强
- 后训练三件套：SFT（监督）→ RLHF / DPO（强化，呼应第 4 章）→ 对齐
- 训练要什么：数据、算力、优化器、分布式（只讲每个是干嘛的，点到为止）
- ▶ 动手：小语料上跑一次"下一个词预测"训练循环，看 loss 往下掉

### 第 8 章　推理 — 概率函数怎么"跑起来"
- 推理 vs 训练：不再更新参数，只前向算概率
- 自回归生成：一步步采样下一个 token
- 采样策略：greedy / temperature / top-p / top-k（回扣第 5 章的概率分布）
- KV cache（加速要点）、上下文窗口、超了怎么办
- 幻觉 / 重复 / "一本正经胡说"——全部回到概率函数解释
- ▶ 动手：实现最简自回归生成循环

## Part C · LLM 工程应用（重心，最长最深；每章回扣原理）

### 第 9 章　提示词工程 — 跟概率函数"说话"
- 为什么 prompt 有效：你在改输入 → 改变条件概率分布
- 指令 / 角色 / few-shot / CoT；为什么 CoT 让模型更准（把隐式推理变成显式 token）
- 提示词的脆弱性与鲁棒写法
- ▶ 动手：同一任务对比 zero / few-shot / CoT 效果差

### 第 10 章　上下文工程 — 往概率函数的"视野"里塞什么
- 上下文工程 ≠ 提示词工程：后者是"说什么"，前者是"窗口里放什么"
- RAG：把外部知识检索进上下文
- 长上下文陷阱：lost-in-the-middle、注意力被稀释
- 上下文压缩、记忆系统（短期 / 长期）
- ▶ 动手：搭一个最简 RAG

### 第 11 章　Harness 工程 — 驱动模型的"骨架"
- 什么是 harness：包裹 LLM 的运行时——系统提示、工具循环、记忆、重试、错误恢复
- **核心澄清：prompt / context / harness / agent 四者的边界**（这四个并列，就是为了分清它们）
- 工具调用机制：模型吐结构化意图 → harness 执行 → 结果回灌上下文
- 循环、状态管理、失败恢复
- ▶ 动手：写一个最简 harness（系统提示 + 工具循环 + 重试）

### 第 12 章　Agent 工程 — 让模型自己朝目标走
- agent = harness + 自主规划 / 循环 / 记忆 / 工具
- 规划：ReAct、反思、任务分解；多 agent 协作（简介）
- 失败模式与评估
- **RL 驱动 agent**（回扣第 4 章）：从规则驱动到学习驱动，agent 自己进化
- ▶ 动手：在第 11 章 harness 上加 ReAct 循环，做个能上网/算数的小 agent

### 第 13 章　收尾 — 把整条线收起来 + 往前看
- 从一个神经元到一个 agent：全书脉络一张大图
- 第一性原理回顾：概率函数 + 参数更新，所有现象从这里出发
- 局限与前沿：幻觉、对齐、效率、agent 可靠性
- 给"自己继续学"的路线图
