# 《从0到1理解AI》内容创作 Spec

> 日期：2026-06-14
> 状态：创作原则已定，待作者审阅
>
> **范围声明**：本 spec 定义"内容怎么写"——创作原则、章节模板、演示的内容侧约定。技术平台、构建、组件基础设施见 [网站架构 spec](./2026-06-14-site-architecture.md)。两者解耦：平台定插槽与契约，内容侧按此填入。本 spec 是**创作原则的唯一来源**；`chapters/outline.md` 只保留章节结构。

---

## 1. 目标与定位
- **读者**：有编程基础、零 AI 基础的自学者（也是作者自学笔记）
- **形态**：可交互的网页教材（章节正文 + 内嵌演示）
- **不是**：纯学术教科书，也不是"30 分钟速成"快餐

## 2. 两个核心创作原则（最高优先级，贯穿全书）
1. **第一性原理**：每个概念追到最底层——它到底是什么、为什么这样设计。具体落点 = 全书两块"骨头"（§4）。
2. **简洁明了 / 高信息密度**：一句话能说清绝不用两句；直觉和例子优先；砍掉一切凑字数的展开。每句话都带新信息。

## 3. 三条基调
1. **深度**：感性理解为主，原理讲透。只用小学数学（加减乘除、平均），**不写矩阵 / 概率 / 求和号 / 偏导**——核心是原理，不是形式。
2. **重心**：原理精炼到"够用"，重点放在 LLM 工程应用（提示词 / 上下文 / harness / agent）。
3. **动手**：每章配一个可交互演示（见 §6）。

## 4. 两块"骨头"（第一性原理的具体落点）
- **骨头 #1：学习 = 参数更新**（损失 → 梯度 → 反向传播 → 参数滑动）
- **骨头 #2：大模型 = "下一个 token 的概率函数"** `f(上下文) → 概率分布`

> 两块都是**锚点章**——讲透后，后续所有现象（生成、采样、幻觉、温度、CoT、上下文）都从它们推导，让学习者"自己想明白为什么"，而不是背结论。

## 5. 章节模板（内容侧，对接架构 spec §4）
- 一个章节 = `chapters/` 下一个 `.md`（命名 `chNN-<slug>.md`）。
  > 注：架构 spec §3 写作 `content/`；本 spec 与现状用 `chapters/`。二选一后同步两边（见 §7）。
- frontmatter 按架构 spec §4.1：`title` / `order` / `part` / `draft`。
- 正文统一三段式：**直觉 →（关键处）极简图示 → 可交互演示**。
- part 分组：`导览` / `原理骨` / `LLM 核心` / `LLM 工程应用`（侧边栏由 config 按 `part` 自动分组）。
- 章节清单、顺序、part 归属在 `chapters/outline.md` 维护；改章节只动它 + 对应 `.md`。

## 6. 演示（"▶ 动手"）的内容侧约定（对接架构 spec §4.2 / §5）
- 每章"动手" = 一个 **Vue 3 SFC 演示**（`<script setup lang="ts">`），自包含、可重置，用 `<DemoFrame>` 挂入（架构 spec §4.3）。
- 按架构 spec §5 两种原型按需选用：
  - **Simulation**（无网络）：纯原理可视化（参数滑动、动画、曲线）—— 原理章（Part A / B）用。
  - **API-backed**（真实模型）：看真实 LLM 响应 —— 工程应用章（Part C）用。
- **全场 TypeScript**（架构 spec §2）。
- 章节正文可展示该演示的关键 TS 逻辑供理解，但"能跑能玩"的体验靠内嵌演示，不是控制台脚本。
- 演示的真实输出/行为**必须实跑验证，不编造**。

## 7. 与架构 spec 的待统一项（需作者拍板）
