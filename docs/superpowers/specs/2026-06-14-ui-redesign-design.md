# 《从0到1理解AI》前端 UI 重设计 Spec —「坐标纸笔记本」

> 日期：2026-06-14
> 状态：方向与令牌已定；本轮实现 **首页 + 第 1 章**。
> 设计宪法：`docs/superpowers/specs/frontend-design.md`——避开 AI 三个默认脸（暖米+衬线+赤陶 / 近黑+酸绿 / 报纸细线），把张扬集中在一处。

---

## 1. 题材锚点

- **题材**：从零讲透的 AI 教学书（《从0到1理解AI》，14 章）。
- **读者**：想真正"懂"的自学者，只用小学数学——不被吓到、也不被当小孩哄。
- **一页一件事**：每章讲透**一个第一性原理** + 一个可玩的演示。站点 = **一本你信得过、又能动手把玩的书**。
- 由此排除：冷企业文档、花哨营销落地页、儿童玩具风。

---

## 2. 设计令牌

### 2.1 调色（每模式 6 个命名色）

**亮（白坐标纸）**：`--paper #F6F7F9` · `--ink #1A1D23` · `--ink-2 #5A6273` · `--rule #E1E4EA` · `--indigo #2D4EF5`（含 hover `#1E40D6`、淡底 `rgba(45,78,245,.12)`）· `--grid rgba(26,29,35,.05)`

**暗（深蓝坐标纸）**：`--paper #0F121A` · `--ink #E6E9EF` · `--ink-2 #9AA3B2` · `--rule #222838` · `--indigo #6E8AFF`（暗底提亮）· `--grid rgba(255,255,255,.045)`

> 亮暗双模式都做（演示已主题感知）。全站**只有一个高饱和色（靛蓝）**，刻意替代 VitePress 默认绿。

### 2.2 字体（两个角色，零网络依赖）

- **正文 `--font-sans`**：系统 CJK 栈 + 系统 Latin（`PingFang SC / Microsoft YaHei / Noto Sans CJK SC / system-ui …`）。零加载，中文渲染交给系统好字。
- **数据/数字/代码/标签 `--font-mono`**：精炼**系统等宽栈**（`JetBrains Mono / Cascadia Code / SF Mono / Consolas …`）+ `font-variant-numeric: tabular-nums`。
- **取舍**：理想用 IBM Plex Mono 做"性格"，但读者在中国、Google Fonts 不稳、自托管需下二进制 → 改系统等宽栈。性格靠**等宽 + tabular-nums + 坐标纸上下文**给，而非特定字体。可自托管时一行升级。
- **标题不加衬线展示字**：CJK 展示字 webfont 数 MB，对静态科普站不值。标题性格靠**等宽章节眉标 + 紧字距 + 坐标纸底**。

### 2.3 签名：坐标纸（全书被记住的那一物）

- 全站 `body` 一层 **24px 方格底**（亮 5% / 暗 4%），静态、不视差、不闪——**就是一张纸**。
- canvas 演示自带的格线对齐同一视觉语言；散点 / 损失曲线 / 神经网络连线**都长在同一张坐标纸上**。
- 长文区让极淡格线均匀透出（5% 对 CJK 阅读不构成干扰）。

### 2.4 字号尺度（type scale）—— 全站唯一来源

所有字号集中为 `--fs-*` 令牌，定义在 `style.css` 的 `:root`，全站与所有章节 / 演示共用——**改字号只动这一处**。各组件与演示用 `var(--fs-*)` 引用，不写裸数值（scoped 样式能读到全局令牌）。

| 令牌 | 值 | 用途 |
|---|---|---|
| `--fs-eyebrow` | `0.86rem` | 等宽小眉标：hero name / 侧边栏 Part / 演示提示 / 曲线说明 |
| `--fs-eyebrow-lg` | `0.98rem` | 大号等宽眉标：信条标题 |
| `--fs-label` | `0.82rem` | 等宽数据标签：按钮 / 章节号 / 参数值 / stat |
| `--fs-meta` | `0.94rem` | 次级正文：地图章节标题 / 演示控件行 |
| `--fs-body` | `17px` | 阅读正文（章节正文列） |
| `--fs-lead` | `1.02rem` | 加重次级：信条正文 |
| `--fs-title` | `1rem` | 小标题：演示框标题 / Part 名 |
| `--fs-h3` / `--fs-h2` / `--fs-h1` | `1.12 / 1.45 / 2.05rem` | 章节标题层级 |
| `--fs-display` | `1.5rem` | 区块大标题：全书地图 |
| `--fs-axis` / `--fs-anno` | `12 / 13px` | SVG 坐标轴 / 标注 |

## 3. 版式

### 3.1 首页（`layout: home` + 两个 slot 注入，不重造轮子）

1. **Hero**：等宽靛蓝眉标（`name`）+ 大字紧字距标题（`text`）+ 论点（`tagline`）+ 靛蓝实心 / 墨色描边两按钮。
2. **`home-hero-after`** → `HomeSketch`：一条**横贯坐标纸的下降 loss 曲线**，标注"学习 = loss 一路下降"。这是首页的论文级图示。
3. **三信条**（features）：去掉默认圆角卡片，改成**顶部细线 + 等宽靛蓝小标题**的编辑式三栏。
4. **`home-features-after`** → `ChapterMap`：**全书地图**——4 个 Part × 14 章在坐标纸上排开，已写的（ch001）靛蓝可点，待写的灰显。这是首页中心件（structure as information），把书的全貌一次给读者。

### 3.2 阅读页（第 1 章）

- 正文行高 **1.85**（CJK 友好）；标题紧字距；`h2` 顶部分隔线，给章节以"分格"节奏。
- 引用块（`>` 第一性原理 / 底座）→ **靛蓝左栏面板**（细线 + paper 底）。
- 行内 `code` → 靛蓝淡底等宽；代码块 → 细线圆角；表格沿用细线。
- **DemoFrame** → 重新皮肤为坐标纸面板（细线、头部分隔、靛蓝描边重置按钮、body 内边距）。
- **LineFitDemo 不改逻辑**：它读 `--vp-c-brand-1`，覆写为靛蓝后拟合线/损失曲线自动变色。

### 3.3 导航 / 侧边栏

- 导航栏**实色 paper 底 + 底细线**（盖住格线，干净）；站名字距收紧。
- 侧边栏 Part 组标题 → 等宽小写处理；激活链接靛蓝（品牌变量自动）。

## 4. 不做（YAGNI）

- 不引入新依赖（字体 / 动画库）。
- 不改 LineFitDemo 已验证的逻辑。
- 不动其它章节（尚无）。
- 不加视差 / 闪烁 / 入场动画——"就是一张纸"。

---

## 5. 健壮性

- **不运行**（CLAUDE.md：用户自跑）。靠 VitePress 默认结构 + CSS 变量覆写 + 文档化 slot，最小化"看不见的破坏面"。
- 用 `home-hero-after` / `home-features-after` 两个**最稳的标准 slot** 注入自定义块（非 `home-hero-image`，避免其渲染条件的不确定）。
- 全局 `prefers-reduced-motion` 降级；亮暗双模式；移动端栅格自降列。
