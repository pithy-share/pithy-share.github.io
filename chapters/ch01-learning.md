# 第 1 章　机器到底怎么"学"

> **第一性原理 #1：学习 = 参数更新。** 所有"机器学习"、所有"训练大模型"，剥到最底层都是同一件事——反复调整参数，让误差变小。这一章把这句话讲透。

## 一句话先抓住本质

人会"学习"：做错了，下次换个做法。机器的学习没这么神秘，它就是一句话：

> **学习 = 调整参数，让误差变小。**

没有魔法。参数是一堆数字，误差用一个公式量出来，然后按一个固定规则改数字，改到误差没法再小为止。

你以后看到任何复杂的词——"训练 GPT"、"微调模型"、"深度学习"——别被吓到，底座都是这一句。区别只在于：参数从 2 个变成几千亿个，"误差"的算法变得更精细。但"调参数降误差"这个动作，一次都没变过。

这一章，我们用最小的例子把这句话走通。

## 把抽象落地：一个最简单的"学习"任务

假设有一堆数据点，横轴 `x`，纵轴 `y`，它们大致排成一条直线。我们不知道这条直线是谁画的，只知道它的样子大概是：

```
y = w · x + b
```

`w` 和 `b` 是两个数字，叫**参数**。现在它们未知——这就是要"学"的东西。

"学"的过程只有四步：

1. 先瞎猜一对 `w`、`b`（比如都填 0）。
2. 用它算出来的直线，跟真实数据比，差多远？
3. 根据差的方向，把 `w`、`b` 调好一点点。
4. 重复，直到差得足够小。

没有任何玄机。下面三小节，就是把这个"调"字拆开。

## 第一步：损失函数——量化"错得多离谱"

要调参数，先得知道现在错成什么样。**损失函数（loss function）** 就是干这个的：把"当前参数有多烂"打成一个分数。

最直观的一种叫**均方误差（MSE）**：对每个点，算"预测值和真实值的差"，平方一下，再求平均。

```
L = (1/n) · Σ (w·xᵢ + b - yᵢ)²
```

两个设计小心思：
- **平方**让正负误差不会互相抵消（错 +3 和错 −3 都算"错 3"，不是"没错"）。
- **平方**还顺带让大错惩罚得更狠（错 2 比两个"错 1"加起来更扎眼）。

分数越高越烂。"学习"的目标，就是让这个分数往下掉。

## 第二步：梯度下降——往下坡走一步

现在要让 `L` 变小，可 `L` 是 `w`、`b` 的函数，怎么知道该往哪调？

想象你蒙着眼站在山上，想走到谷底。你能感到脚下哪边是下坡——那就往下坡迈一步。**梯度（gradient）** 就是"脚下的坡度"：它告诉你，`L` 沿着参数方向哪边在涨、涨得多快。

走法自然就是：**往梯度相反的方向走**（梯度指向"涨"，反方向才是"下坡"）。

```
w ← w − 学习率 · (L 对 w 的梯度)
b ← b − 学习率 · (L 对 b 的梯度)
```

对 1D 直线 + MSE，梯度直接能算出来：

```
∂L/∂w = (2/n) · Σ xᵢ · (w·xᵢ + b − yᵢ)
∂L/∂b = (2/n) · Σ       (w·xᵢ + b − yᵢ)
```

不用记公式，记住意思就行：**梯度 = "每个参数现在该往哪调、调多少"的指示牌**。

**学习率（learning rate）** 是每步走多大。太大，一步跨过谷底飞到对面山坡，永远下不去；太小，磨磨蹭蹭。这是要调的旋钮，但不是玄学，就一个数。

> 三步连起来：**算损失 → 算梯度 → 反方向走一小步**。循环往复，损失一路下滑，参数一路滑向"对的位置"。这就是"学习"的全部机制。

## 第三步：反向传播——链式法则分摊"责任"

上面 `y = w·x + b` 只有两层，梯度一眼能算。但真正的模型是**一长串运算**：输入经过一个又一个步骤，最后才得到预测值。这种时候，每个参数的梯度怎么算？

答案就是微积分里最朴素的**链式法则**：如果 A 影响 B、B 影响 C，那 A 对 C 的影响 = (A 对 B 的影响) × (B 对 C 的影响)。一层乘一层。

举个最小链条，损失要经过三步才碰到参数 `w`：

```
z = w·x + b        ← 参数 w 在这一步出场
â = σ(z)           ← 再过一道非线性（下一章细讲）
L = (â − y)²       ← 最后才算损失
```

要算 `∂L/∂w`，把它拆成三段局部梯度乘起来：

```
∂L/∂w = ∂L/∂â · ∂â/∂z · ∂z/∂w
         最后一步   中间一步   w 直接出场那一步
```

每一小步的梯度都好算（都是简单微积分），乘起来就是 `w` 对最终损失的影响。

**反向传播（backpropagation）** 就是这个过程：从损失出发，**从后往前**，一层一层算局部梯度、乘上去，把"这个误差该怪哪个参数"的责任分摊到每个参数头上。链条越长，能分摊的参数越多。当链条变成几十上百层——那就是下一章的**神经网络**。机制没变，还是这条链式法则。

> 关键直觉：反向传播不是神经网络专属的新东西，它就是链式法则。链式法则是数学事实，跟模型长什么样无关。

## 动手：亲眼看参数自己滑到对的位置

下面这段代码把三步全走一遍：造一批满足 `y = 2x + 3` 的带噪声数据，让机器从 `w=0, b=0` 开始，自己学出这条直线。

```python
import random
random.seed(0)            # 固定随机种子，你跑出来的数和我一模一样

# 1. 造数据：真实关系 y = 2x + 3，加点噪声，假装我们不知道
true_w, true_b = 2.0, 3.0
xs = [0.1 * i for i in range(-20, 21)]                  # x 从 -2.0 到 2.0
ys = [true_w * x + true_b + random.gauss(0, 0.2)
      for x in xs]

# 2. 参数随便初始化（"还没学"的状态）
w, b = 0.0, 0.0
lr = 0.01            # 学习率：每步走多大

# 3. 反复：算损失 → 算梯度 → 反方向走一步
for step in range(1000):
    grad_w = sum(2 * (w*x + b - y) * x for x, y in zip(xs, ys)) / len(xs)
    grad_b = sum(2 * (w*x + b - y)       for x, y in zip(xs, ys)) / len(xs)
    w -= lr * grad_w
    b -= lr * grad_b

    if step % 200 == 0:
        loss = sum((w*x + b - y)**2 for x, y in zip(xs, ys)) / len(xs)
        print(f"step {step:4d}  w={w:.3f}  b={b:.3f}  loss={loss:.4f}")

print(f"学到的直线：y = {w:.3f}·x + {b:.3f}")
```

跑出来是这样（固定了随机种子，你跑出来会一模一样）：

```
step    0  w=0.056  b=0.060  loss=13.9584
step  200  w=2.006  b=2.933  loss=0.0513
step  400  w=2.012  b=2.983  loss=0.0485
step  600  w=2.012  b=2.984  loss=0.0485
step  800  w=2.012  b=2.984  loss=0.0485
学到的直线：y = 2.012·x + 2.984
```

盯住 `w` 和 `b`：它们从 0 出发，自己一步步滑向 `2` 和 `3`——而我们从没告诉过它答案。**这就是"学习"在发生。** `loss` 从约 14 一路掉到 0.05，几乎不再下降，说明参数已经到了"谷底"。

动手改两个地方感受一下：
- 把 `lr` 调到 `0.05` 看它更快收敛；调到 `0.5` 看它震荡甚至飞掉（一步跨过了谷底）。
- 把 `range(1000)` 改小，看"没学够"时长什么样。

## 收束：这就是所有"学习"的底座

整章压缩成一行：

> **学习 = 用损失函数量化误差，用梯度下降调参数，必要时用反向传播算梯度。**

把这句话刻在脑子里。下一章我们把"两个参数"换成"成千上万个参数组成的网络"；第 7 章我们把"均方误差"换成"预测下一个词的错误率"去训练大模型——但机制，一次都没换过。
