# 第 1 章　机器到底怎么"学"

> **第一性原理 #1：学习 = 参数更新。** 所有机器学习、所有大模型训练，剥到底都是同一件事：调参数，让误差变小。这一章只讲清楚这一件事。

## 学习是什么：调旋钮，让误差变小

想象一台机器有一堆**旋钮**（参数）。给它一些题，它答得有对有错。"学习"就是：看看错多少，把旋钮往让错误变小的方向拧一点，再答，再拧……拧到错误没法再小。

就这么一件事。你以后看到"训练 GPT""微调模型""深度学习"，别被吓到——区别只是旋钮从几个变成几千亿个、题目从算术变成预测文字，"拧旋钮降误差"这个动作从没变过。这一章用最小的例子把它走通。

## 损失：把"错得多离谱"变成一个数

光说"错了"没用，得量化成数字，这个数叫**损失**。

最简单的算法：每个点，算"预测值和真实值差多少"，把这个差**平方**一下（这样不管多算还是少算，都变正，不会正负抵消），再把所有点**平均**。比如预测 5、实际 3，差是 2，平方是 4；预测 1、实际 3，差是 −2，平方也是 4——错得一样狠，扣一样多分。这个平均分就是损失，**越小越好**。

## 梯度：往下坡走一步

损失会随每个旋钮变化。**梯度**就是"脚下的坡度"——它指向损失涨得最快的方向（最陡的上坡）。想降损失，自然就反着它走，往下坡迈一小步。每个旋钮都这么拧。

每轮拧多大，叫**学习率**：拧太大，一步跨过最优值、来回乱晃；拧太小，磨磨蹭蹭。就一个要调的数。

反复"算损失 → 看梯度 → 往下坡拧一小格"，损失一路下滑，旋钮一路滑到最优位置。**这就是学习的全部动作。**

## 反向传播：错误怎么一步步追责到每个旋钮

只有一两个旋钮时，梯度一眼能看。但真实模型里，一个旋钮要经过一长串计算才影响最终答案——像多米诺骨牌，一棒传一棒。这时怎么知道最前面那个旋钮该为最后的错误担多少责？

靠的是常识：**影响是连着的，一路乘起来**。A 影响 B，B 影响 C，那 A 对 C 的影响，就是把 A→B 和 B→C 这两段的传递乘在一起。

**反向传播**就干这个：从最后的错误出发，**从后往前**，每经过一道计算就把"这段传了多少影响"乘一次，一路追到每个旋钮——"这个错，怪这个旋钮多少"。旋钮多、链条长，这游戏就多做几遍。下一章把旋钮堆成几十上百层的**神经网络**，机制还是这个，没换过。

## 动手：看旋钮自己滑到对的位置

造一批满足 `y = 2x + 3` 的带噪声数据点，从旋钮全归零开始，让机器自己把这条直线学出来（JavaScript，浏览器里直接能跑）。

```js
// 固定种子的随机数发生器（让每次跑结果都一样），细节不用管
function makeRand(seed) {
  return () => (seed = (seed * 1664525 + 1013904223) % 4294967296) / 4294967296;
}
const rand = makeRand(1);

const trueW = 2.0, trueB = 3.0;
const xs = Array.from({ length: 41 }, (_, i) => 0.1 * (i - 20));   // x: -2.0 ~ 2.0
const ys = xs.map(x => trueW * x + trueB + (rand() - 0.5) * 0.4);  // 加点噪声

let w = 0, b = 0, lr = 0.01;
for (let step = 0; step < 1000; step++) {
  // 两个旋钮各自的"坡度"：误差乘上各自的份量，求平均
  let gradW = 0, gradB = 0;
  for (let i = 0; i < xs.length; i++) {
    const err = w * xs[i] + b - ys[i];
    gradW += 2 * err * xs[i];
    gradB += 2 * err;
  }
  gradW /= xs.length; gradB /= xs.length;
  w -= lr * gradW;   // 往下坡方向拧 w 一小格
  b -= lr * gradB;   // 拧 b 一小格
  if (step % 200 === 0) {
    let loss = 0;
    for (let i = 0; i < xs.length; i++) loss += (w * xs[i] + b - ys[i]) ** 2;
    loss /= xs.length;
    console.log(`step ${step}  w=${w.toFixed(3)}  b=${b.toFixed(3)}  loss=${loss.toFixed(4)}`);
  }
}
console.log(`学到的直线：y = ${w.toFixed(3)}·x + ${b.toFixed(3)}`);
```

```
step 0  w=0.057  b=0.060  loss=14.1163
step 200  w=2.022  b=2.951  loss=0.0147
step 400  w=2.028  b=3.002  loss=0.0119
step 600  w=2.028  b=3.003  loss=0.0119
step 800  w=2.028  b=3.003  loss=0.0119
学到的直线：y = 2.028·x + 3.003
```

`w`、`b` 两个旋钮从 0 自己拧到了 `2` 和 `3`——我们从没告诉它答案，**这就是学习在发生**。把 `lr` 调到 `0.5`，看它一步跨过最优值、来回震荡。

## 底座

> 学习 = 用损失量化误差 + 按梯度往下坡拧旋钮 + 用反向传播算每个旋钮该拧多少。
